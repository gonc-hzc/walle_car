# PID_TEST 接口与接线文档

本文档按当前工程 `PID_TEST.ioc`、`Core/Src/motor.c`、`Core/Src/encoder.c`、`Core/Src/bluetooth.c`、`Core/Src/remote_control.c` 重新整理。重点覆盖电机、编码器、蓝牙、调试串口、SPI2 预留接口，以及左右速度不一致和起步反向的接线排查。

## 1. 必查结论

左右速度不同、刚推左摇杆时短暂后退，既可能来自软件，也可能来自接线。接线侧最常见的是:

- 左右电机输出接反: 左右通道互换，或某个电机 M+/M- 反了。
- 编码器 A/B 相接反: 速度符号变反，PID 会越调越偏。
- 左右编码器接到对方通道: 左轮转动却改变右轮计数。
- 编码器电平不匹配: 5 V 编码器输出直接进 STM32，可能造成误触发或损坏。
- TB6612 STBY 未拉高: 电机不转、偶发抖动或看似控制异常。
- 所有电源没有共地: PWM、串口、编码器信号都可能异常。

当前代码定义: 正 `throttle` 表示前进；正 `turn` 表示右转。混控为:

| 项目 | 计算 |
| --- | --- |
| 左目标 | `throttle + turn` |
| 右目标 | `throttle - turn` |

如果 `throttle > 0` 且 `turn = 0`，两侧目标应相同，车应直行前进。若某一侧后退，优先查电机方向线和编码器方向。

## 2. 电源与公共地

| 模块 | 连接 | 说明 |
| --- | --- | --- |
| STM32 | 3.3 V / GND | STM32F103C8T6 逻辑电平为 3.3 V |
| TB6612 VCC | 3.3 V | 逻辑电源 |
| TB6612 VM | 电机电源正极，例如 12 V | 不要接 STM32 3.3 V |
| TB6612 GND | 系统 GND | 与 STM32、电机电源、编码器、蓝牙共地 |
| TB6612 STBY | 3.3 V | 当前工程没有分配 STBY 控制脚，必须外部拉高 |
| 编码器 VCC | 按编码器规格，常见 5 V | A/B 输出若为 5 V，需降到 3.3 V |
| BLE 模块 VCC | 按模块规格，常见 5 V | 模块 TX 若为 5 V，也需降到 3.3 V |

注意: STM32 输入脚不能长期直接承受 5 V TTL。PA0、PB0、PB1、PB10、PA3 若外部模块输出 5 V，建议使用分压、电平转换，或确认模块输出实际为 3.3 V。

## 3. TB6612 电机接口

### STM32 到 TB6612

| 功能 | STM32 引脚 | TB6612 引脚 | 代码通道 |
| --- | --- | --- | --- |
| 左电机 PWM | PA6 | PWMA | TIM3_CH1 |
| 左电机方向 1 | PB8 | AIN1 | `Motor_Left_Forward()` 置 1 |
| 左电机方向 2 | PB7 | AIN2 | `Motor_Left_Backward()` 置 1 |
| 右电机 PWM | PA7 | PWMB | TIM3_CH2 |
| 右电机方向 1 | PB6 | BIN1 | `Motor_Right_Forward()` 置 1 |
| 右电机方向 2 | PA5 | BIN2 | `Motor_Right_Backward()` 置 1 |
| 公共地 | GND | GND | 必须共地 |

### TB6612 到电机

| TB6612 输出 | 推荐连接 | 说明 |
| --- | --- | --- |
| AO1 | 左电机一端 | 左电机输出 |
| AO2 | 左电机另一端 | 左电机输出 |
| BO1 | 右电机一端 | 右电机输出 |
| BO2 | 右电机另一端 | 右电机输出 |

如果某个电机正转方向与车辆前进方向相反，优先交换该电机的两根动力线，例如 AO1/AO2 或 BO1/BO2。交换动力线后，还要重新确认编码器速度符号。

### 方向真值表

| 电机 | 状态 | IN1 | IN2 | PWM |
| --- | --- | --- | --- | --- |
| 左电机 | 前进 | PB8=1 | PB7=0 | PA6 输出占空比 |
| 左电机 | 后退 | PB8=0 | PB7=1 | PA6 输出占空比 |
| 左电机 | 停止/滑行 | PB8=0 | PB7=0 | PA6=0 |
| 右电机 | 前进 | PB6=1 | PA5=0 | PA7 输出占空比 |
| 右电机 | 后退 | PB6=0 | PA5=1 | PA7 输出占空比 |
| 右电机 | 停止/滑行 | PB6=0 | PA5=0 | PA7=0 |

PWM 参数:

| 定时器 | 通道 | 引脚 | Prescaler | Period | 频率 | 占空比范围 |
| --- | --- | --- | --- | --- | --- | --- |
| TIM3 | CH1 | PA6 | 71 | 999 | 约 1 kHz | 0-999 |
| TIM3 | CH2 | PA7 | 71 | 999 | 约 1 kHz | 0-999 |

## 4. 编码器接口

工程使用外部中断读取编码器。每一路只在 A 相上升沿中断，读取 B 相判断方向。

| 功能 | STM32 引脚 | 外部连接 | 配置 |
| --- | --- | --- | --- |
| 左编码器 A 相 | PA0-WKUP | 左电机编码器 A | EXTI0，上升沿，内部上拉 |
| 左编码器 B 相 | PB0 | 左电机编码器 B | GPIO 输入，内部上拉 |
| 右编码器 A 相 | PB1 | 右电机编码器 A | EXTI1，上升沿，内部上拉 |
| 右编码器 B 相 | PB10 | 右电机编码器 B | GPIO 输入，内部上拉 |
| 编码器 VCC | 5 V 或编码器规格电压 | 编码器电源 | 注意信号电平 |
| 编码器 GND | GND | 系统公共地 | 必须共地 |

当前代码计数方向:

| 编码器 | A 相触发 | B 相电平 | 计数变化 |
| --- | --- | --- | --- |
| 左编码器 | PA0 上升沿 | PB0=1 | `+1` |
| 左编码器 | PA0 上升沿 | PB0=0 | `-1` |
| 右编码器 | PB1 上升沿 | PB10=1 | `-1` |
| 右编码器 | PB1 上升沿 | PB10=0 | `+1` |

右编码器方向判定与左编码器相反，这是当前代码为了左右对称安装而做的约定。如果你的实际安装不同，可能需要交换编码器 A/B 相，或修改 `Core/Src/encoder.c` 的计数符号。

中断优先级:

| 中断 | 来源 | 优先级 |
| --- | --- | --- |
| EXTI0_IRQn | PA0，左编码器 A 相 | 1 |
| EXTI1_IRQn | PB1，右编码器 A 相 | 1 |

## 5. 串口与蓝牙

### USART1: USB-TTL 调试与有线遥控

| 功能 | STM32 引脚 | 外部连接 | 参数 |
| --- | --- | --- | --- |
| USART1_TX | PA9 | USB-TTL RXD | 115200, 8N1 |
| USART1_RX | PA10 | USB-TTL TXD | 115200, 8N1 |
| GND | GND | USB-TTL GND | 必须共地 |

USART1 支持完整 JSON 遥控命令，例如:

```json
{"throttle":500,"turn":0}
```

### USART2: BLE 蓝牙串口

| 功能 | STM32 引脚 | BLE 模块 | 参数 |
| --- | --- | --- | --- |
| USART2_TX | PA2 | BLE RXD | 9600, 8N1 |
| USART2_RX | PA3 | BLE TXD | 9600, 8N1 |
| GND | GND | BLE GND | 必须共地 |
| VCC | 5 V 或模块规格电压 | BLE VCC | 按模块要求 |

BLE 当前优先支持短 JSON，例如:

```json
{"v":500,"w":0}
```

其中 `v` 对应 `throttle`，`w` 对应 `turn`。桥接程序默认发送 `minjson`，与 STM32 当前代码匹配。不要使用 `compact` 格式，除非 STM32 端增加了 `Txxx,Rxxx` 解析器。

## 6. SPI2 预留接口

`.ioc` 当前已启用 SPI2 主机，并配置了两个片选脚。若暂时不用，可以悬空片选外设端，但 STM32 引脚仍按当前工程初始化。

| 功能 | STM32 引脚 | 外部建议连接 | 配置 |
| --- | --- | --- | --- |
| SPI2_SCK | PB13 | SPI SCK | Mode master |
| SPI2_MISO | PB14 | SPI MISO | Mode master |
| SPI2_MOSI | PB15 | SPI MOSI | Mode master |
| GYRO CS | PB11 | BMI088_GYRO_CS1 | GPIO 输出，初始高 |
| ACC CS | PB12 | BMI088_ACC_CS2 | GPIO 输出，初始高 |

SPI2 参数: 主机模式，全双工，CPOL=High，CPHA=2Edge，分频 32，约 1.125 Mbit/s。

## 9. 上车前接线排查流程

按下面顺序查，能最快区分是线错、方向错，还是 PID 参数问题。

### 9.1 不开 PID，先查电机方向

临时只给单个电机固定 PWM，确认方向:

| 测试 | 期望 |
| --- | --- |
| 左电机正 PWM | 左轮向车辆前进方向转 |
| 右电机正 PWM | 右轮向车辆前进方向转 |
| 左电机负 PWM | 左轮向车辆后退方向转 |
| 右电机负 PWM | 右轮向车辆后退方向转 |

如果某一侧方向反了，交换该侧电机动力线，或修改 `Motor_*_Forward()` / `Motor_*_Backward()` 的方向 GPIO。

### 9.2 手动转轮，查编码器归属

断开电机动力或让小车离地，手动转动轮子:

| 操作 | 期望 |
| --- | --- |
| 只转左轮 | 只有左编码器计数变化 |
| 只转右轮 | 只有右编码器计数变化 |

如果转左轮变右计数，说明左右编码器线接反。先修归属，再看方向。

### 9.3 正转时查速度符号

让单个电机以低 PWM 前进，观察速度:

| 操作 | 期望 |
| --- | --- |
| 左轮前进 | `left_speed` 为正 |
| 右轮前进 | `right_speed` 为正 |

如果前进时速度为负，说明该侧编码器方向与代码相反。可交换该侧编码器 A/B 相，或修改 `encoder.c` 的加减号。

### 9.4 查左右速度差

在低速固定 PWM 下比较左右:

| 现象 | 优先原因 |
| --- | --- |
| 一边为正、一边为负 | 编码器方向或电机方向接反 |
| 两边都正，但差很多 | 电机/减速箱差异、轮胎摩擦、驱动压降、编码器漏脉冲 |
| 速度跳变明显 | 编码器信号电平、接地、A/B 干扰、线太长 |
| 推前进先后退 | 前端首包、摇杆坐标、或电机/编码器方向不一致 |

### 9.5 恢复 PID 后再调参数

确认电机方向、编码器归属、速度符号都正确后，再调 PID。否则 PID 会把接线错误放大，表现为突然反转、速度越调越偏、左右 PWM 饱和。

## 10. 推荐接线标签

实际制作线束时建议贴下面这些标签:

| 标签 | 连接 |
| --- | --- |
| L_PWM | PA6 -> TB6612 PWMA |
| L_IN1 | PB8 -> TB6612 AIN1 |
| L_IN2 | PB7 -> TB6612 AIN2 |
| R_PWM | PA7 -> TB6612 PWMB |
| R_IN1 | PB6 -> TB6612 BIN1 |
| R_IN2 | PA5 -> TB6612 BIN2 |
| L_ENC_A | PA0 -> 左编码器 A |
| L_ENC_B | PB0 -> 左编码器 B |
| R_ENC_A | PB1 -> 右编码器 A |
| R_ENC_B | PB10 -> 右编码器 B |
| BLE_TX | PA3 <- BLE TXD |
| BLE_RX | PA2 -> BLE RXD |
| GND | 全系统公共地 |
